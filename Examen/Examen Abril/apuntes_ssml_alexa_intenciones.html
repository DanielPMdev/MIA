SSML · Alexa · Intenciones

Apuntes para el examen — etiquetas de voz, componentes del asistente, intenciones y slots, Lambda y costes AWS.

SSML es un lenguaje de marcado para controlar cómo se sintetiza la voz. Todo el contenido va dentro de <speak>. En el examen te dan la chuleta con etiquetas, atributos y valores — aquí aprendes para qué sirve cada una.

1 <break> — Pausas

Inserta una pausa en la voz. Tiene dos formas: por tiempo exacto o por intensidad.

<break time="5s"/>   ← pausa de 5 segundos
<break strength="strong"/>  ← pausa larga sin tiempo fijo

Valores de strength: none · x-weak · weak · medium · strong · x-strong

2 <lang> — Cambio de idioma

Hace que una palabra o frase se pronuncie en otro idioma, usando el motor de voz correspondiente.

El <lang xml:lang="en-US">streamer</lang> vasco...

Valores comunes: es-ES · es-MX · en-US · en-GB · fr-FR · de-DE

3 <emphasis> — Énfasis

Destaca una palabra o frase cambiando su entonación y ritmo. Solo tiene un atributo: level.

<emphasis level="strong">las Campanadas</emphasis>
<emphasis level="reduced">desvela quién</emphasis>

Valores: strong · moderate · reduced

4 <p> y <s> — Párrafo y sentencia

<p> introduce una pausa larga entre párrafos (como un punto y aparte). <s> introduce una pausa de sentencia (como un punto seguido más controlado).

<p>Primer párrafo.</p>
<p><s>Una frase concreta.</s></p>

5 <prosody> — Volumen, velocidad y tono

La etiqueta más importante. Controla tres parámetros de la voz. Se pueden anidar para combinar efectos.

Volumen: <prosody volume="x-loud">...</prosody>
Velocidad: <prosody rate="slow">...</prosody>
Tono: <prosody pitch="x-high">...</prosody>
Duración: <prosody duration="20s">...</prosody>
volume
silent · x-soft · soft · medium · loud · x-loud · +3dB / -2dB
rate
x-slow · slow · medium · fast · x-fast · 20% · 70%
pitch
x-low · low · medium · high · x-high · -10% · +5%
duration
20s · 500ms — fuerza la duración total del fragmento
Anidación: puedes meter un <prosody> dentro de otro. El interior hereda el contexto del exterior y añade su propio modificador encima.
Anidación doble (tono + velocidad):
<prosody pitch="x-low">
  texto normal con tono bajo
  <prosody rate="x-fast">texto rápido Y con tono bajo</prosody>
</prosody>

6 <say-as> — Interpretación especial

Le dice al motor cómo debe interpretar el texto: como fecha, fracción, número, dirección, etc.

<say-as interpret-as="date" format="dmy">11-05-1978</say-as>
<say-as interpret-as="fraction">1/3</say-as>
<say-as interpret-as="cardinal">42</say-as>

interpret-as: character · cardinal · ordinal · digits · fraction · unit · date · time · address · telephone

format (para date): mdy · dmy · ymd · md · dm · d · m · y

7 <amazon:effect name="whispered"> — Susurro

Exclusivo de Alexa/Amazon Polly. Hace que el texto se pronuncie en modo susurro.

<amazon:effect name="whispered">texto en susurro</amazon:effect>

ASR — Automatic Speech Recognition El oído

Recibe las ondas de sonido de tu voz y las convierte en texto escrito. Sin ASR, el bot solo escucharía ruido.

Entrada: Señal de audio  |  Salida: Texto transcrito

Ejemplo: dices "¿Qué tiempo hace en Valencia?" → el ASR escribe exactamente eso.

NLU — Natural Language Understanding El cerebro

Toma el texto y extrae dos cosas fundamentales: la intención (qué quiere el usuario) y las entidades / slots (datos concretos que necesita).

Entrada: Texto  |  Salida: Intención + Slots

Ejemplo: "¿Qué tiempo hace en Valencia?" → Intención: Consultar_clima · Slot ciudad: Valencia · Slot fecha: Ahora

DM — Dialog Manager El estratega

Decide qué hacer con la intención detectada. Si faltan slots, pregunta al usuario (Elicit Slot). Si tiene todo, ejecuta la acción (Fulfillment). Gestiona el flujo de la conversación.

Entrada: Contexto (intención + slots)  |  Salida: Acción o pregunta

Ejemplo: tiene "Consultar_clima" y "Valencia" → busca los datos meteorológicos y genera el texto de respuesta.

TTS — Text-to-Speech La garganta artificial

Convierte el texto de respuesta generado por el DM en voz sintetizada usando voces grabadas o de IA. Aquí es donde entra SSML.

Entrada: Texto  |  Salida: Voz

Ejemplo: "En Valencia hace 22 grados y está soleado" → lo dice en voz alta con la voz de Alexa.

Elicit Slot

El bot detecta la intención pero le falta un slot obligatorio. Le pregunta al usuario para completarlo.

"¿Para qué ciudad quieres el tiempo?" → le falta el slot Ciudad.

Fulfillment

El bot tiene todos los slots que necesita y ejecuta la acción: llama a una API, hace la reserva, etc.

Tiene Ciudad:Valencia → llama a la API del tiempo → responde con los datos.

Tres ejemplos del flujo completo

"¿Qué tiempo hace en Valencia?"

ASR → texto · NLU → Consultar_clima + Ciudad:Valencia + Fecha:Ahora · DM → llama a API del tiempo · TTS → "Hacen 20° y está soleado"

"¿Cuáles son las noticias del día?"

ASR → texto · NLU → Obtener_noticias + Fecha:Hoy · DM → busca noticias o pregunta por temas · TTS → lee los titulares

"Pon una alarma para las 7 de la mañana"

ASR → texto · NLU → Crear_alarma + Hora:07:00 · DM → crea alarma, pregunta si repetir · TTS → "Alarma programada para las 7"

La intención (intent) es el objetivo del usuario. Los slots son las variables concretas que necesitas para cumplirlo. Es como una función: la intención es el nombre, los slots son los parámetros.
Truco de examen: la intención responde a "¿qué quiere hacer?" · los slots responden a "¿con qué datos?"
Frase del usuario Intención Slots
"Quiero reservar una mesa para cuatro personas mañana a las dos." Hacer_Reserva Personas: 4 Fecha: mañana Hora: 14:00
"¿Tenéis sitio para cenar hoy a las nueve?" Consultar_Disponibilidad Fecha: hoy Hora: 21:00 Tipo: cena
"Me gustaría cancelar mi reserva del viernes." Cancelar_Reserva Fecha: viernes
"Cambia mi reserva de las 8 por una a las 10 de la noche." Modificar_Reserva Hora_original: 20:00 Hora_nueva: 22:00

Elicit Slot vs Fulfillment

Elicit Slot

El bot detecta la intención pero le falta un slot obligatorio. Le pregunta al usuario para completarlo.

"¿Para qué ciudad quieres el tiempo?" → le falta el slot Ciudad.

Fulfillment

El bot tiene todos los slots que necesita y ejecuta la acción: llama a una API, hace la reserva, etc.

Tiene Ciudad:Valencia → llama a la API del tiempo → responde con los datos.
Cuando el bot tiene la intención y todos los slots, necesita conectarse al mundo real para obtener o guardar datos. Eso lo hace una función AWS Lambda.

Flujo técnico — bot del tiempo

1NLU (Lex) detecta la intención Consultar_Clima + Ciudad: Valencia
2Lex envía un archivo JSON con esos datos a la función Lambda
3Lambda llama a la API de meteorología y obtiene "22°C y soleado"
4TTS (Polly) convierte la respuesta en voz para el usuario

Costes AWS — modelo de facturación

Amazon Lex
Se paga por utterance — cada vez que el usuario habla al bot
$0,004 / solicitud
Amazon Polly (TTS)
Se paga por caracteres convertidos a voz
$4,00 / millón chars
Amazon Transcribe (ASR)
Se paga por minutos de audio procesados
$0,024 / minuto
Ejemplo del examen: 1.000 solicitudes → Lex $4,00 · 200.000 chars → Polly $0,80 · 100 min audio → Transcribe $2,40 = Total: $7,20/mes
